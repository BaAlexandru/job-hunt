Create `.github/workflows/ci.yml` with the following exact structure:

**Top-level configuration:**
- `name: CI`
- Trigger: `on: push: branches: [master]`
- Concurrency: `group: ci-${{ github.ref }}`, `cancel-in-progress: true`
- Permissions: `contents: read`, `packages: write`

**Job 1: `test-backend`** (runs-on: ubuntu-latest)
- Service containers (matching compose.yaml versions and test application.yml credentials):
  - `postgres`: image `postgres:17`, env `POSTGRES_DB=jobhunt`, `POSTGRES_USER=jobhunt`, `POSTGRES_PASSWORD=jobhunt`, port 5432:5432, health-cmd `pg_isready`, interval 10s, timeout 5s, retries 5
  - `redis`: image `redis:7-alpine`, port 6379:6379, health-cmd `redis-cli ping`, interval 10s, timeout 5s, retries 5
  - `minio`: image `minio/minio:latest`, port 9000:9000, env `MINIO_ROOT_USER=minioadmin`, `MINIO_ROOT_PASSWORD=minioadmin`, health-cmd `curl -f http://localhost:9000/minio/health/live || exit 1`, interval 10s, timeout 5s, retries 10. NOTE: Do NOT use `mc ready local` (mc CLI not available in the minio/minio image in GH Actions service containers).
- Steps: checkout@v5, setup-java@v4 (distribution: temurin, java-version: '24'), gradle/actions/setup-gradle@v4, then `run: ./gradlew :backend:test`
- No SMTP service container needed -- EmailService has try/catch graceful degradation and email tests use MockK
- **MinIO fallback:** If the `minio/minio:latest` default CMD does not start the S3 server (i.e., the health check never passes), remove the `minio` service container and replace it with a pre-step: `docker run -d --name minio -p 9000:9000 -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin minio/minio server /data` followed by a health wait loop: `until curl -sf http://localhost:9000/minio/health/live; do sleep 1; done`

**Job 2: `test-frontend`** (runs-on: ubuntu-latest)
- No service containers needed (pure Vitest unit/component tests)
- Steps: checkout@v5, pnpm/action-setup@v4, setup-node@v4 (node-version: '22', cache: pnpm, cache-dependency-path: frontend/pnpm-lock.yaml), `run: pnpm --dir frontend install --frozen-lockfile`, `run: pnpm --dir frontend test:ci`

**Job 3: `build-push`** (runs-on: ubuntu-latest, needs: [test-backend, test-frontend])
- Steps:
  - checkout@v5
  - docker/setup-buildx-action@v3
  - docker/login-action@v3 with registry: ghcr.io, username: `${{ github.repository_owner }}`, password: `${{ secrets.GITHUB_TOKEN }}`
  - **Backend metadata:** docker/metadata-action@v5, id: meta-backend, images: `ghcr.io/baalexandru/jobhunt-backend`, flavor: `latest=false`, tags: `type=sha` + `type=raw,value={{date 'YYYYMMDD' tz='UTC'}},enable={{is_default_branch}}` + `type=raw,value=latest,enable={{is_default_branch}}`
  - **Backend build:** docker/build-push-action@v6, context: `.`, file: `./backend/Dockerfile`, push: true, tags: `${{ steps.meta-backend.outputs.tags }}`, labels: `${{ steps.meta-backend.outputs.labels }}`, cache-from: `type=gha,scope=backend`, cache-to: `type=gha,scope=backend,mode=max`
  - **Frontend metadata:** docker/metadata-action@v5, id: meta-frontend, images: `ghcr.io/baalexandru/jobhunt-frontend`, flavor: `latest=false`, tags: same pattern as backend
  - **Frontend build:** docker/build-push-action@v6, context: `./frontend`, file: `./frontend/Dockerfile`, push: true, tags: `${{ steps.meta-frontend.outputs.tags }}`, labels: `${{ steps.meta-frontend.outputs.labels }}`, build-args: `NEXT_PUBLIC_API_URL=https://job-hunt.dev/api` (per Claude's discretion: single domain with Traefik routing, build-time baking is appropriate), cache-from: `type=gha,scope=frontend`, cache-to: `type=gha,scope=frontend,mode=max`

**Job 4: `scan`** (runs-on: ubuntu-latest, needs: [build-push])
- Steps:
  - docker/login-action@v3 with registry: ghcr.io, username: `${{ github.repository_owner }}`, password: `${{ secrets.GITHUB_TOKEN }}` (needed to pull recently-pushed images)
  - **Scan backend:** aquasecurity/trivy-action@0.33.1 (verify tag exists at https://github.com/aquasecurity/trivy-action/releases before implementation; if not, use the latest 0.x release), scan-type: image, image-ref: `ghcr.io/baalexandru/jobhunt-backend:latest`, format: table, output: trivy-backend.txt, severity: CRITICAL,HIGH,MEDIUM, exit-code: '0' (report only, never fail)
  - **Scan frontend:** aquasecurity/trivy-action@0.33.1, scan-type: image, image-ref: `ghcr.io/baalexandru/jobhunt-frontend:latest`, format: table, output: trivy-frontend.txt, severity: CRITICAL,HIGH,MEDIUM, exit-code: '0'
  - **Summary step** (if: always()): Append scan results to `$GITHUB_STEP_SUMMARY` with markdown headers "## Vulnerability Scan Results", "### Backend Image", code block with `cat trivy-backend.txt 2>/dev/null || echo "Scan output not available"`, "### Frontend Image", code block with `cat trivy-frontend.txt 2>/dev/null || echo "Scan output not available"`
