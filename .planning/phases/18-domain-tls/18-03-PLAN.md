**Pre-requisite check:** Ensure the user has created `infra/tofu/main/terraform.tfvars` (gitignored) with:
```
cloudflare_api_token = "..."   # Cloudflare API token scoped to Zone:DNS:Edit, Zone:Settings:Edit, Zone:SSL and Certificates:Edit
cloudflare_zone_id   = "..."   # Cloudflare zone ID for job-hunt.dev
```

If the file does not exist, STOP and ask the user to create it. Do NOT proceed without these values.

**Step 1: Initialize and apply OpenTofu**
```bash
cd infra/tofu/main
tofu init -upgrade    # Downloads cloudflare and tls providers
tofu plan             # Review changes -- should show ~15+ new resources
tofu apply            # Creates DNS records, Origin CA cert, zone settings, SG update
```

Verify: `tofu output origin_ca_cert` should return a non-empty value (redacted). DNS records should start resolving within minutes.

**Step 2: Seal Origin CA cert into K8s secrets**

Ensure kubectl is connected to the cluster (via `infra/scripts/connect.sh` SSH tunnel if needed).

```bash
bash infra/scripts/seal-origin-ca.sh
```

This creates:
- `infra/k8s/traefik/origin-ca-sealed-secret.yaml` (cluster-wide scope for kube-system)
- `infra/k8s/argocd/argocd-tls-sealed-secret.yaml` (namespace-scoped for argocd)

**Step 3: Apply K8s manifests**

Apply in order:

```bash
# SealedSecrets first (so the TLS secrets exist before Traefik/ArgoCD reference them)
kubectl apply -f infra/k8s/traefik/origin-ca-sealed-secret.yaml
kubectl apply -f infra/k8s/argocd/argocd-tls-sealed-secret.yaml

# Verify secrets were unsealed
kubectl get secret origin-ca-tls -n kube-system
kubectl get secret argocd-server-tls -n argocd

# Traefik configuration
kubectl apply -f infra/k8s/traefik/tls-store.yaml
kubectl apply -f infra/k8s/traefik/tls-option.yaml
kubectl apply -f infra/k8s/traefik/traefik-config.yaml

# Wait for Traefik to restart (HelmChartConfig triggers Helm upgrade)
sleep 10
kubectl get pods -n kube-system -l app.kubernetes.io/name=traefik

# ArgoCD IngressRouteTCP
kubectl apply -f infra/k8s/argocd/ingress-route-tcp.yaml

# Update ArgoCD Application specs (ignoreDifferences)
kubectl apply -f infra/argocd/apps/prod.yaml
kubectl apply -f infra/argocd/apps/staging.yaml
```

**Step 4: Change ArgoCD admin password**
```bash
bash infra/scripts/change-argocd-password.sh
```
The user must enter a strong password and store it in their password manager.

**Step 5: Scale up staging for verification**
```bash
bash infra/scripts/staging-up.sh
```

**Step 6: Run automated verification**
```bash
bash infra/scripts/verify-phase18.sh
```

All checks should pass. If any fail, debug based on the specific failure (DNS propagation delay, Traefik pod not restarted, SealedSecret not unsealed, etc.).

**Step 7: Scale staging back down**
```bash
bash infra/scripts/staging-down.sh
```
