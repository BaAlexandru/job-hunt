<objective>
Create the operational tooling for data stores: MinIO bucket initialization Job, daily PostgreSQL backup CronJob, and a convenience script for downloading backups locally.

Purpose: MinIO needs buckets created before the application can use it (jobhunt-documents for file uploads, jobhunt-backups for database dumps). Automated daily backups protect against data loss. The download script enables the user to archive backups locally before the 7-day MinIO lifecycle expiry.

Output: Bucket init Job manifest, backup CronJob manifest, updated base kustomization, download-backups.sh script.
</objective>

1. Create `infra/k8s/base/minio/bucket-init-job.yaml` with this exact content:
```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: minio-bucket-init
  labels:
    app: minio-bucket-init
    app.kubernetes.io/part-of: jobhunt
spec:
  backoffLimit: 5
  template:
    metadata:
      labels:
        app: minio-bucket-init
    spec:
      restartPolicy: OnFailure
      automountServiceAccountToken: false
      containers:
        - name: mc
          image: minio/mc:RELEASE.2025-09-07T16-10-49Z
          command: ["/bin/sh", "-c"]
          args:
            - |
              mc alias set myminio http://minio:9000 $MINIO_ROOT_USER $MINIO_ROOT_PASSWORD &&
              mc mb myminio/jobhunt-documents --ignore-existing &&
              mc mb myminio/jobhunt-backups --ignore-existing &&
              mc ilm rule ls myminio/jobhunt-backups --json | grep -q '"expiration"' ||
              mc ilm rule add myminio/jobhunt-backups --expire-days 7
          envFrom:
            - secretRef:
                name: minio-secrets
          resources:
            requests:
              memory: 32Mi
              cpu: 25m
            limits:
              memory: 64Mi
              cpu: 100m
```

Key details:
- Image pinned to `minio/mc:RELEASE.2025-09-07T16-10-49Z` (last available release)
- Uses `--expire-days` (NOT `--expiry-days` -- verified against official MinIO docs)
- `restartPolicy: OnFailure` so it retries if MinIO is not yet ready
- `backoffLimit: 5` gives sufficient retries for MinIO startup delay
- References `minio-secrets` which contains `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD`
- MinIO service accessible at `http://minio:9000` within the namespace

2. Create `infra/k8s/base/postgres/backup-cronjob.yaml` with this exact content:
```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: postgres-backup
  labels:
    app: postgres-backup
    app.kubernetes.io/part-of: jobhunt
spec:
  schedule: "0 2 * * *"
  concurrencyPolicy: Forbid
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 3
      template:
        metadata:
          labels:
            app: postgres-backup
        spec:
          restartPolicy: OnFailure
          automountServiceAccountToken: false
          containers:
            - name: backup
              image: postgres:17
              command: ["/bin/sh", "-c"]
              args:
                - |
                  TIMESTAMP=$(date +%Y%m%d-%H%M%S)
                  FILENAME="jobhunt-${TIMESTAMP}.sql.gz"
                  echo "Starting backup: ${FILENAME}"
                  pg_dump -h postgres -U "$POSTGRES_USER" -d "$POSTGRES_DB" --format=plain | gzip > /tmp/${FILENAME} &&
                  curl -sSL https://dl.min.io/client/mc/release/linux-amd64/mc -o /usr/local/bin/mc &&
                  chmod +x /usr/local/bin/mc &&
                  mc alias set myminio http://minio:9000 "$MINIO_ROOT_USER" "$MINIO_ROOT_PASSWORD" &&
                  mc cp /tmp/${FILENAME} myminio/jobhunt-backups/${FILENAME} &&
                  echo "Backup complete: ${FILENAME}"
              env:
                - name: PGPASSWORD
                  valueFrom:
                    secretKeyRef:
                      name: postgres-secrets
                      key: POSTGRES_PASSWORD
              envFrom:
                - secretRef:
                    name: postgres-secrets
                - secretRef:
                    name: minio-secrets
              resources:
                requests:
                  memory: 64Mi
                  cpu: 50m
                limits:
                  memory: 128Mi
                  cpu: 250m
```

Key details:
- Schedule `0 2 * * *` = daily at 02:00 UTC
- `concurrencyPolicy: Forbid` prevents overlapping runs
- Uses `postgres:17` image (same as database for pg_dump compatibility)
- Downloads `mc` binary at runtime (~25MB static binary, fast download)
- Plain format pg_dump piped through gzip for human-readable compressed output
- `PGPASSWORD` from `postgres-secrets.POSTGRES_PASSWORD` for non-interactive pg_dump auth
- `POSTGRES_USER` and `POSTGRES_DB` injected from `postgres-secrets` via envFrom
- `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` injected from `minio-secrets` via envFrom
- Connects to postgres at `postgres:5432` (headless service DNS) and minio at `minio:9000`

Create `infra/scripts/download-backups.sh` with this content:
```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"

NAMESPACE="${1:-jobhunt-prod}"
LOCAL_BACKUP_DIR="${2:-$PROJECT_ROOT/backups}"
mkdir -p "$LOCAL_BACKUP_DIR"

# Read EC2 connection details from tofu outputs
ELASTIC_IP=$(tofu -chdir="$PROJECT_ROOT/infra/tofu/main" output -raw elastic_ip)
SSH_KEY=$(tofu -chdir="$PROJECT_ROOT/infra/tofu/main" output -raw ssh_private_key_path)

# Verify mc CLI is installed locally
if ! command -v mc &> /dev/null; then
  echo "Error: mc (MinIO Client) is not installed locally."
  echo "Install: https://min.io/docs/minio/linux/reference/minio-mc.html"
  exit 1
fi

# Verify MINIO_ROOT_USER and MINIO_ROOT_PASSWORD are set
if [[ -z "${MINIO_ROOT_USER:-}" ]] || [[ -z "${MINIO_ROOT_PASSWORD:-}" ]]; then
  echo "Error: MINIO_ROOT_USER and MINIO_ROOT_PASSWORD must be set."
  echo "Export them before running this script."
  exit 1
fi

echo "Setting up port-forward to MinIO in ${NAMESPACE}..."
ssh -L 9000:localhost:9000 -i "$SSH_KEY" -o StrictHostKeyChecking=accept-new \
  ubuntu@"$ELASTIC_IP" \
  "kubectl port-forward -n ${NAMESPACE} svc/minio 9000:9000" &
SSH_PID=$!
trap "kill $SSH_PID 2>/dev/null || true" EXIT
sleep 3

echo "Configuring mc alias..."
mc alias set k8s-minio http://localhost:9000 "$MINIO_ROOT_USER" "$MINIO_ROOT_PASSWORD"

echo "Downloading backups to ${LOCAL_BACKUP_DIR}..."
mc mirror k8s-minio/jobhunt-backups "$LOCAL_BACKUP_DIR/"

echo ""
echo "Backups downloaded to $LOCAL_BACKUP_DIR"
echo "Files:"
ls -lh "$LOCAL_BACKUP_DIR"
```

Then make it executable:
```bash
chmod +x infra/scripts/download-backups.sh
```

Also add `backups/` to the project `.gitignore` if not already present (downloaded backups should not be committed).

Key details:
- Follows same pattern as `connect.sh`: reads tofu outputs for ELASTIC_IP and SSH_KEY
- First arg overrides namespace (default: `jobhunt-prod`), second arg overrides local dir (default: `PROJECT_ROOT/backups`)
- Validates `mc` CLI is installed locally before attempting
- Validates `MINIO_ROOT_USER` and `MINIO_ROOT_PASSWORD` env vars are set
- Uses `mc mirror` for incremental sync (skips already-downloaded files)
- SSH tunnel forwards MinIO port 9000 through the EC2 instance
- Trap ensures SSH tunnel is cleaned up on exit
