Create directory `infra/k8s/namespaces/`.

**namespaces.yaml** — Two namespace definitions in a single file separated by `---`:
```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: jobhunt-staging
  labels:
    environment: staging
    app.kubernetes.io/part-of: jobhunt
---
apiVersion: v1
kind: Namespace
metadata:
  name: jobhunt-prod
  labels:
    environment: production
    app.kubernetes.io/part-of: jobhunt
```

**limitrange-staging.yaml** — LimitRange for staging namespace:
```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
  namespace: jobhunt-staging
spec:
  limits:
    - type: Container
      default:
        memory: 256Mi
        cpu: 250m
      defaultRequest:
        memory: 128Mi
        cpu: 100m
      max:
        memory: 512Mi
        cpu: "1"
```

**limitrange-prod.yaml** — Identical LimitRange for prod namespace:
```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
  namespace: jobhunt-prod
spec:
  limits:
    - type: Container
      default:
        memory: 256Mi
        cpu: 250m
      defaultRequest:
        memory: 128Mi
        cpu: 100m
      max:
        memory: 512Mi
        cpu: "1"
```

These are kept OUTSIDE the Kustomize overlays to avoid the namespace transformer gotcha (Kustomize would incorrectly apply overlay namespace to cluster-scoped Namespace resources). They are applied directly with `kubectl apply -f infra/k8s/namespaces/`.
