Create three shell scripts. All scripts use `#!/usr/bin/env bash` with `set -euo pipefail`. All read infrastructure values dynamically:
```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
ELASTIC_IP=$(tofu -chdir="$PROJECT_ROOT/infra/tofu/main" output -raw elastic_ip)
SSH_KEY=$(tofu -chdir="$PROJECT_ROOT/infra/tofu/main" output -raw ssh_private_key_path)
```

**bootstrap-k3s.sh** — SSHes into EC2 and installs K3s:
- SSH into ubuntu@$ELASTIC_IP using $SSH_KEY with `-o StrictHostKeyChecking=accept-new`
- Run K3s install remotely: `ssh ... "curl -sfL https://get.k3s.io | K3S_KUBECONFIG_MODE='644' sh -"`
- Wait for node ready: `ssh ... "sudo k3s kubectl wait --for=condition=Ready node --all --timeout=120s"`
- Print success message with instructions to run `setup-kubeconfig.sh` next
- Do NOT manage swap (Phase 14 user-data handles it)
- Do NOT open port 6443 (SSH tunnel only)

**setup-kubeconfig.sh** — One-time kubeconfig fetch and merge:
- SCP kubeconfig: `scp -i "$SSH_KEY" -o StrictHostKeyChecking=accept-new ubuntu@"$ELASTIC_IP":/etc/rancher/k3s/k3s.yaml "$PROJECT_ROOT/k3s-kubeconfig.yaml"`
- Verify server URL is `https://127.0.0.1:6443` (it should be by default for SSH tunnel usage)
- Create `~/.kube` directory if not exists
- If `~/.kube/config` exists, merge using `KUBECONFIG=~/.kube/config:$PROJECT_ROOT/k3s-kubeconfig.yaml kubectl config view --flatten > ~/.kube/config.merged && mv ~/.kube/config.merged ~/.kube/config`
- If `~/.kube/config` does not exist, copy directly: `cp "$PROJECT_ROOT/k3s-kubeconfig.yaml" ~/.kube/config`
- Clean up temp file: `rm -f "$PROJECT_ROOT/k3s-kubeconfig.yaml"`
- Rename context to `jobhunt-k3s`: `kubectl config rename-context default jobhunt-k3s`
- Print success message reminding user to run `connect.sh` before kubectl commands

**connect.sh** — Daily-use SSH tunnel for kubectl:
- Print: `"Opening SSH tunnel to $ELASTIC_IP for kubectl (port 6443)..."`
- Print: `"Press Ctrl+C to close the tunnel."`
- Run: `ssh -L 6443:localhost:6443 -i "$SSH_KEY" -o StrictHostKeyChecking=accept-new ubuntu@"$ELASTIC_IP" -N`

Make all three scripts executable: `chmod +x infra/scripts/*.sh`

**staging-up.sh** — Scale staging namespace to replicas=1:
```bash
#!/usr/bin/env bash
set -euo pipefail

NAMESPACE="jobhunt-staging"

echo "Scaling up staging namespace ($NAMESPACE)..."

kubectl scale -n "$NAMESPACE" deploy --all --replicas=1
kubectl scale -n "$NAMESPACE" statefulset --all --replicas=1

echo "Waiting for all pods to be Ready (timeout: 120s)..."
kubectl wait -n "$NAMESPACE" --for=condition=Ready pod --all --timeout=120s

echo ""
echo "Staging pods:"
kubectl get pods -n "$NAMESPACE"
```

**staging-down.sh** — Scale staging namespace to replicas=0:
```bash
#!/usr/bin/env bash
set -euo pipefail

NAMESPACE="jobhunt-staging"

echo "Scaling down staging namespace ($NAMESPACE)..."

kubectl scale -n "$NAMESPACE" deploy --all --replicas=0
kubectl scale -n "$NAMESPACE" statefulset --all --replicas=0

echo "Waiting for pods to terminate..."
kubectl wait -n "$NAMESPACE" --for=delete pod --all --timeout=60s 2>/dev/null || true

echo ""
echo "Staging pods (should be empty):"
kubectl get pods -n "$NAMESPACE"
```

Make both executable: `chmod +x infra/scripts/staging-up.sh infra/scripts/staging-down.sh`
