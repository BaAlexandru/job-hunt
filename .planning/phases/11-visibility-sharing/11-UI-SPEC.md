## Interaction Contracts

### I1: Set Visibility (VISI-01)

**Where:** Company/Job detail or edit page
**Component:** VisibilityControl (Select dropdown)
**Options:** Private (default), Public, Shared
**Behavior:**
- Selecting PUBLIC triggers ConfirmDialog with variant="default"
- Selecting PRIVATE or SHARED applies immediately (optimistic update)
- Select shows current visibility value
- Mutation invalidates the resource detail query key AND browse query keys

**Confirmation dialog (PUBLIC only):**
- Title: "Make publicly visible?"
- Description: "This will make {resource name} visible to all logged-in users."
- Action label: "Make Public"
- Cancel label: "Cancel"

### I2: Share by Email (VISI-02)

**Where:** Company/Job detail page, ShareManager section (visible only when visibility is SHARED or as a standalone section below visibility control)
**Behavior:**
1. Owner types email into Input field
2. Owner clicks "Share" button (primary variant)
3. On success: email clears, new share appears in list below, toast notification "Shared with {email}"
4. On error (user not found): toast error "No account found for {email}"
5. On error (self-share): toast error "You cannot share with yourself"
6. On error (duplicate): toast error "Already shared with {email}"

**Share list:**
- Each row shows: user email + "Shared {relative time}" + Revoke button (ghost variant, destructive text)
- Revoke triggers ConfirmDialog: title "Revoke access?", description "Remove {email}'s access to {resource name}?", action label "Revoke", variant="destructive"
- Empty share list shows inline text: "Not shared with anyone yet"

### I3: Browse Public Resources (VISI-03)

**Where:** `/browse` page
**Layout:** Tabs component with "Companies" and "Jobs" tabs
**Card layout:** Responsive grid, same as existing companies/jobs pages (`grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3`)
**BrowseCard content:**
- Resource name (title)
- Key metadata (location for companies, company name for jobs)
- "Shared by {owner email}" in `text-xs text-muted-foreground`
- Globe icon badge in card header
- Card is clickable, navigates to read-only detail view
- No edit/delete actions in dropdown (view-only)

**Empty state:**
- Heading: "No public resources yet"
- Body: "When users share companies or jobs publicly, they will appear here."

### I4: Shared With Me (VISI-04)

**Where:** `/shared` page
**Layout:** Tabs component with "Companies" and "Jobs" tabs
**Card layout:** Same responsive grid as browse page
**Card content:**
- Uses existing CompanyCard/JobCard but without edit/delete dropdown menu items
- Shows Users icon badge via VisibilityBadge
- Card is clickable, navigates to read-only detail view

**Empty state:**
- Heading: "Nothing shared with you"
- Body: "When someone shares a company or job with you, it will appear here."

### I5: Visibility Badge on Cards (VISI-01)

**Where:** Existing CompanyCard and JobCard components, card header area
**Behavior:**
- PRIVATE: no icon displayed (default, no visual change)
- PUBLIC: Globe icon, `size-3.5 text-muted-foreground`, aria-label="Public"
- SHARED: Users icon, `size-3.5 text-muted-foreground`, aria-label="Shared"
- Position: Left of the card title, inline
- Non-interactive (no click handler, informational only)

### I6: Read-Only Detail View (VISI-05)

**Where:** Detail page for any resource viewed by a non-owner (shared or public)
**Behavior:**
- All data fields are displayed (same as owner view)
- Edit button: hidden
- Delete button: hidden
- Visibility control: hidden
- Share manager: hidden
- Applications/interviews/documents tabs: hidden (these are strictly private per CONTEXT.md)
- Show a subtle banner at top: "You are viewing this as a shared resource" or "You are viewing a public resource"

**Banner spec:**
- Full-width, `bg-muted` background, `text-sm text-muted-foreground`, `px-4 py-2`
- Icon: Eye icon from lucide-react, `size-4`, inline before text
