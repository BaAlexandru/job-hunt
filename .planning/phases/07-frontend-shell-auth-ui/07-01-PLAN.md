POST /api/auth/register  { email: string, password: string } -> 201 { message: string }
POST /api/auth/login     { email: string, password: string } -> 200 { accessToken: string, expiresIn: number, tokenType: string } + Set-Cookie: refresh_token (httpOnly, path=/api/auth/refresh)

POST /api/auth/logout    (Authorization header + cookie)     -> 200 { message: string } + clear refresh_token cookie
GET  /api/auth/verify?token={token}                          -> 200 { message: string }
POST /api/auth/password-reset       { email: string }        -> 200 { message: string }
POST /api/auth/password-reset/confirm { token: string, newPassword: string } -> 200 { message: string }

Error response format: { status: number, error: string, message: string, path: string, timestamp: string, fieldErrors?: Record<string, string> }

CORS: localhost:3000 allowed with credentials

2. Create `frontend/lib/validators.ts` with Zod schemas matching backend validation rules:
   - `loginSchema`: email (valid email), password (non-empty string)
   - `registerSchema`: email (valid email), password (min 8 chars, regex `^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).+$` with descriptive error messages)
   - `passwordResetSchema`: email (valid email)
   - `passwordResetConfirmSchema`: token (non-empty), newPassword (same rules as register password)

3. Create `frontend/lib/api-client.ts`:
   - Module-level `let accessToken: string | null = null`
   - Export `setAccessToken(token: string | null)` function
   - Export `ApiError` class extending Error with `status: number` and `data: ErrorResponse` fields
   - Export `async function apiClient<T>(path: string, options?: RequestInit): Promise<T>`:
     - Base URL from `process.env.NEXT_PUBLIC_API_URL` falling back to `"http://localhost:8080/api"`
     - Set `Content-Type: application/json` header
     - Attach `Authorization: Bearer {accessToken}` when accessToken is not null
     - Always use `credentials: "include"` (sends refresh_token cookie cross-origin)
     - On 401 response when accessToken exists: call `refreshAccessToken()`, if success retry the original request with new token, if fail throw ApiError
     - On non-ok response: parse JSON body, throw `new ApiError(response.status, errorData)`
     - On 204 response: return `undefined as T`
     - On success: return `response.json()`
   - Internal `async function refreshAccessToken(): Promise<boolean>`:
     - POST to `${API_BASE}/auth/refresh` with `credentials: "include"` (no body, no auth header needed -- cookie carries refresh token)
     - On success: set `accessToken = data.accessToken`, return true
     - On failure: set `accessToken = null`, return false

4. Create `frontend/lib/auth-context.tsx` ("use client" component):
   - Interface `User { email: string }`
   - Interface `AuthContextType { user: User | null, isLoading: boolean, login(email, password): Promise<void>, register(email, password): Promise<void>, logout(): Promise<void> }`
   - `AuthProvider` component:
     - State: `user` (User | null), `isLoading` (boolean, initially true)
     - On mount (useEffect): attempt session restore by calling `refreshAccessToken()` via the api-client module. If refresh succeeds, decode user email from JWT payload (base64 decode the middle segment, parse JSON, read `sub` claim which contains the email). Set user. Set isLoading to false.
     - `login(email, password)`: call `apiClient<AuthResponse>("/auth/login", { method: "POST", body: JSON.stringify({ email, password }) })`, call `setAccessToken(data.accessToken)`, decode email from JWT `sub` claim, set user, set a non-httpOnly cookie `has_session=true` with path `/` (for proxy.ts route protection)
     - `register(email, password)`: call `apiClient<MessageResponse>("/auth/register", { method: "POST", body: JSON.stringify({ email, password }) })`, return (no auto-login, user must verify email first)
     - `logout()`: call `apiClient("/auth/logout", { method: "POST" }).catch(() => {})`, call `setAccessToken(null)`, set user to null, delete `has_session` cookie
     - Export `useAuth()` hook that reads context and throws if used outside AuthProvider
