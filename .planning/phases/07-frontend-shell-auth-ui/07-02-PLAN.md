1. Create `frontend/hooks/use-auth.ts` ("use client") with TanStack Query mutation hooks:
   - `useLogin()`: useMutation calling `useAuth().login(data.email, data.password)`. On success: `toast.success("Signed in successfully")`, redirect to `/dashboard` via `useRouter().push("/dashboard")`. On error: if ApiError with status 401 show `toast.error("Incorrect email or password")`, else `toast.error("Connection failed. Please try again.")`.
   - `useRegister()`: useMutation calling `useAuth().register(data.email, data.password)`. On success: `toast.success("Account created. Check your email to verify.")`, redirect to `/login`. On error: if ApiError with status 409 show `toast.error("An account with this email already exists. Try signing in instead.")`, else generic error toast.
   - `usePasswordReset()`: useMutation calling `apiClient<MessageResponse>("/auth/password-reset", { method: "POST", body: JSON.stringify(data) })`. On success: `toast.success("Reset link sent to your email")`.
   - `usePasswordResetConfirm()`: useMutation calling `apiClient<MessageResponse>("/auth/password-reset/confirm", { method: "POST", body: JSON.stringify(data) })`. On success: `toast.success("Password reset successfully")`, redirect to `/login`.

2. Create `frontend/components/auth/login-form.tsx` ("use client"):
   - Use `useForm<LoginInput>` with `zodResolver(loginSchema)` and defaultValues `{ email: "", password: "" }`
   - Use `useLogin()` mutation hook
   - Render shadcn `<Form>` with two `<FormField>` elements:
     - Email: `<Input placeholder="you@example.com" type="email" />`
     - Password: `<Input type="password" />`
   - Submit button: `<Button type="submit" className="w-full" disabled={loginMutation.isPending}>`
     - Text: `loginMutation.isPending ? "Signing in..." : "Sign in"`
   - Below form: Link text "Don't have an account? Create one" linking to `/register`
   - Below that: Link text "Forgot your password?" linking to `/forgot-password`

3. Create `frontend/components/auth/register-form.tsx` ("use client"):
   - Use `useForm<RegisterInput>` with `zodResolver(registerSchema)` and defaultValues `{ email: "", password: "" }`
   - Use `useRegister()` mutation hook
   - Render shadcn `<Form>` with two `<FormField>` elements (same pattern as login)
   - Submit button: text `isPending ? "Creating account..." : "Create account"`
   - Below form: "Already have an account? Sign in" linking to `/login`

4. Create `frontend/app/(auth)/layout.tsx`:
   - Server component (no "use client")
   - Centered card layout: `<div className="flex min-h-screen items-center justify-center bg-background p-4">{children}</div>`
   - This layout wraps all auth pages but does NOT include the sidebar/topbar

     - Loading (no token or request in flight): heading "Verify your email", body with Loader2 spinner
     - Success: heading "Verify your email", body "Email verified successfully. You can now sign in." + Link button to `/login`
     - Error: heading "Verify your email", body "This verification link is invalid or has expired." + Link to `/login`
   - If no token in URL: show "We sent a verification link to your email. Check your inbox." (this page is also shown after registration redirect)

2. Create `frontend/app/(auth)/forgot-password/page.tsx` ("use client"):
   - Render Card with heading "Reset your password"
   - Body text: "Enter your email and we'll send you a reset link."
   - Form with single email field using `useForm` with `zodResolver(passwordResetSchema)`
   - Submit button: "Send reset link"
   - Use `usePasswordReset()` mutation from hooks/use-auth.ts
   - On success: show success message "Check your email for a password reset link." (replace form with message) + Link back to `/login`
   - Below form: "Back to sign in" link to `/login`

3. Create `frontend/app/(auth)/reset-password/page.tsx` ("use client"):

   - Submit button: "Reset password"
   - Use `usePasswordResetConfirm()` mutation, passing `{ token, newPassword }` on submit
   - On success (handled by hook): redirects to /login with toast "Password reset successfully"
