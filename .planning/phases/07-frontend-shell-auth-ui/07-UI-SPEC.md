## Spacing Scale

Declared values (multiples of 4, using Tailwind v4 default spacing):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps, inline padding |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing, input padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing, card padding |
| lg | 24px | `gap-6`, `p-6` | Section padding, sidebar padding, main content padding |
| xl | 32px | `gap-8`, `p-8` | Auth card internal padding, layout gaps |
| 2xl | 48px | `gap-12`, `p-12` | Page-level vertical rhythm |
| 3xl | 64px | `gap-16`, `p-16` | Landing page hero spacing |

Exceptions:
- Sidebar width: 256px (w-64) on desktop, full-width sheet on mobile
- Top bar height: 56px (h-14) -- touch-friendly and visually balanced
- Nav item touch target: 40px minimum height (py-2 on text-sm with gap-3 icons)

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 14px | 500 (medium) | 1.5 | `text-sm font-medium leading-normal` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

**Rationale:** 14px body is the Linear/Notion standard for information-dense UIs. 3 sizes (14, 20, 28) keep the type scale minimal. Two weights only (400, 500, 600) -- medium reserved for labels/nav items, semibold for headings.

**Font loading:**
```typescript
// app/layout.tsx
import { Inter } from "next/font/google"
const inter = Inter({ subsets: ["latin"], variable: "--font-inter" })
```

## Color

shadcn/ui default theme uses OKLCH colors in Tailwind v4. The contract uses shadcn semantic tokens, not raw hex values.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--card` | Page background, card surfaces, input backgrounds |
| Secondary (30%) | `--muted` / `--sidebar` / `--border` | Sidebar background, nav hover states, section dividers, top bar |
| Accent (10%) | `--primary` | See reserved list below |
| Destructive | `--destructive` | Delete actions, error states only |

**Accent reserved for (exhaustive list):**
- Primary CTA buttons ("Sign in", "Create account")
- Active sidebar nav item background
- Form focus ring color
- Link text on hover

**Accent is NOT used for:** icons in nav (use `--muted-foreground`), card borders (use `--border`), section headings (use `--foreground`).

**Dark mode:** Handled automatically by shadcn/ui + next-themes. The `class` strategy on `<html>` tag with `defaultTheme="system"` and `enableSystem`. No manual toggle (deferred per CONTEXT.md).

**Source:** CONTEXT.md locked "neutral defaults, subtle borders, light shadows, whitespace (Linear/Notion-inspired)". shadcn default theme matches this aesthetic exactly.

## Layout Contract

### Auth Pages Layout
- Centered vertically and horizontally on the viewport
- Card container: max-width 400px, padding 32px (p-8)
- Background: `--background` (dominant surface)
- Card: `--card` with `--border` and subtle shadow (`shadow-sm`)

### Dashboard Layout
- Full viewport height (`h-screen`)
- Sidebar: 256px wide (`w-64`), left-aligned, `--sidebar` background, `--border` right border
- Top bar: full width of remaining space, 56px height (`h-14`), `--background` with `--border` bottom border
- Main content: flex-1 with overflow-auto, padding 24px (`p-6`)

### Responsive Breakpoints

| Breakpoint | Width | Behavior |
|------------|-------|----------|
| Mobile | < 768px (md) | Sidebar hidden, hamburger menu icon in top bar, Sheet component slides from left |
| Desktop | >= 768px (md) | Sidebar visible, no hamburger icon |

**Source:** CONTEXT.md locked "sidebar collapses to hamburger slide-over menu on mobile". md (768px) is the standard Tailwind breakpoint for this pattern.

## Interaction States

| Element | Default | Hover | Active/Pressed | Disabled | Focus |
|---------|---------|-------|----------------|----------|-------|
| Primary button | `--primary` bg, `--primary-foreground` text | Slight opacity reduction (`opacity-90`) | Scale down (`scale-[0.98]`) | 50% opacity, cursor-not-allowed | Ring 2px `--ring` |
| Input field | `--background` bg, `--border` border | `--border` darkens slightly | N/A | 50% opacity | Ring 2px `--ring`, border `--ring` |
| Nav item (inactive) | `--muted-foreground` text, transparent bg | `--accent` bg, `--accent-foreground` text | N/A | N/A | Ring 2px `--ring` |
| Nav item (active) | `--accent` bg, `--accent-foreground` text | N/A (already highlighted) | N/A | N/A | Ring 2px `--ring` |
| Link text | `--primary` color | Underline | N/A | N/A | Ring 2px `--ring` |

**Loading states:**
- Button: Text changes to loading copy (e.g., "Signing in..."), button disabled, no spinner icon
- Page transitions: No skeleton in Phase 7 -- use simple centered spinner (lucide-react `Loader2` with `animate-spin`)
- Auth state loading: Full-page centered spinner while `isLoading` is true in auth context
