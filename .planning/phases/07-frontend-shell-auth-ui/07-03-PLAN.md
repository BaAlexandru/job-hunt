1. Create `frontend/proxy.ts` at the frontend project root (NOT inside app/):
   - Import `getSessionCookie` from `better-auth/cookies` for checking Better Auth's session cookie
   - Import `NextRequest`, `NextResponse` from `next/server`
   ```typescript
   import { NextRequest, NextResponse } from "next/server"
   import { getSessionCookie } from "better-auth/cookies"

   const publicPaths = ["/", "/login", "/register", "/verify", "/forgot-password", "/reset-password"]

   export function proxy(request: NextRequest) {
     const { pathname } = request.nextUrl

     // Allow public paths
     if (publicPaths.some(p => pathname === p || pathname.startsWith(p + "/"))) {
       return NextResponse.next()
     }

     // Allow Next.js internals and static assets
     if (pathname.startsWith("/_next") || pathname.startsWith("/favicon") || pathname.startsWith("/api/auth")) {
       return NextResponse.next()
     }

     // Check for Better Auth session cookie
     const sessionCookie = getSessionCookie(request)

     if (!sessionCookie) {
       return NextResponse.redirect(new URL("/login", request.url))
     }

     return NextResponse.next()
   }

   export const config = {
     matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.png$).*)"],
   }
   ```

2. IMPORTANT: The function MUST be named `proxy` (not `middleware`). The file MUST be named `proxy.ts` (not `middleware.ts`). This is a Next.js 16 breaking change from Next.js 15.

3. IMPORTANT: Allow `/api/auth` paths through (Better Auth's API routes must be accessible without a session).

4. NOTE: `getSessionCookie` from `better-auth/cookies` checks for the existence of Better Auth's session cookie. This is a lightweight check (cookie existence only, no server-side validation). This replaces the manual `has_session` cookie from the original plan -- Better Auth manages its own session cookie automatically when users sign in/out.

5. If `getSessionCookie` is not available or doesn't work with the request object in proxy.ts, fall back to checking for Better Auth's default cookie name directly:
   ```typescript
   const hasSession = request.cookies.has("better-auth.session_token") || request.cookies.has("__Secure-better-auth.session_token")
   ```
   Better Auth uses `better-auth.session_token` in development and `__Secure-better-auth.session_token` in production (HTTPS).
