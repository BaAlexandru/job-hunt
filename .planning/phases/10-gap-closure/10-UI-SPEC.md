## Interaction Contracts

### Expandable Row Pattern (shared by GAP-01 and GAP-02)

Both interview notes and document versions use the same expandable row pattern for visual consistency:

| Property | Value |
|----------|-------|
| Trigger | Click anywhere on the row (entire row is clickable) |
| Expand indicator | Lucide `ChevronRight` icon, rotates 90deg clockwise when expanded |
| Animation | CSS transition on chevron rotation: `transition-transform duration-200` |
| Expanded content border | `border-l-2 border-muted` on the left side of expanded content |
| Expanded content indent | `ml-4 pl-4` (16px margin-left + 16px padding-left) |
| Expanded content bottom padding | `pb-4` (16px) |
| Collapse behavior | Click same row to collapse; click different row to collapse current and expand new (single-expand) |
| State management | `useState<string | null>(null)` tracking expanded item ID |

### Inline Edit (GAP-01 notes)

| Property | Value |
|----------|-------|
| Trigger | Click on note content text to enter edit mode |
| Component | Textarea (auto-height) |
| Save trigger | onBlur (save-on-blur, matching QuickNotes pattern) |
| Cancel | Press Escape to revert to server value |
| Loading indicator | Textarea disabled + opacity-50 during mutation |
| Success feedback | sonner toast "Note updated" |

### Note Creation (GAP-01)

| Property | Value |
|----------|-------|
| Layout | Inline form at bottom of expanded notes section |
| Fields | Textarea for content + Select dropdown for note type |
| Note type default | GENERAL (preselected) |
| Submit | Button "Add Note" (primary variant) |
| Clear after submit | Yes -- clear textarea, reset type to GENERAL |
| Validation | Content must not be empty (disable button when empty) |

### Version Upload (GAP-02)

| Property | Value |
|----------|-------|
| Layout | Inline dropzone at bottom of expanded version list |
| Dropzone | react-dropzone with border-dashed, same style as existing document-upload.tsx |
| Fields | File dropzone + Input for optional version note |
| Submit trigger | Automatic on file drop (same as existing document upload pattern) |
| Progress indicator | Spinner icon in upload area during mutation |
| File restrictions | Inherit from existing document upload (no additional restrictions) |

### Delete Interactions

| Property | GAP-01 (Note) | GAP-02 (Version) |
|----------|---------------|------------------|
| Trigger | Trash2 icon button on note row | Trash2 icon button on version row |
| Confirmation | ConfirmDialog with destructive variant | ConfirmDialog with destructive variant |
| Disabled state | Never (all notes deletable) | When only 1 version remains (versions.length <= 1) |
| Success feedback | sonner toast "Note deleted" | sonner toast "Version deleted" |

### Loading States

| Component | Loading State |
|-----------|--------------|
| Notes list (on expand) | Skeleton: 2 lines of `Skeleton` component (h-4 w-full) |
| Version list (on expand) | Skeleton: 3 rows of `Skeleton` component (h-8 w-full) |
| Note save (inline edit) | Textarea disabled + reduced opacity |
| Version upload | Spinner icon replacing dropzone content |
| Set as Current | Button shows Loader2 spinner, disabled during mutation |
