Modify `document-list.tsx` to add an expandable version history panel. The current file renders documents in a DataTable. The version panel must NOT modify the DataTable component itself -- instead, build a custom list rendering that wraps each document row with expandable content.

**Implementation approach:**
Replace the DataTable usage with a custom list that supports expandable rows. Add state `useState<string | null>(null)` for tracking which document is expanded.

**State and hooks to add at the component level:**
- `const [expandedDocId, setExpandedDocId] = useState<string | null>(null)` -- single-expand behavior
- Toggle function: `const toggleExpand = (id: string) => setExpandedDocId(prev => prev === id ? null : id)`

**For each document row:**
- Make the entire row clickable to toggle expand
- Add a `ChevronRight` icon (from lucide-react) that rotates 90deg when expanded: `className="transition-transform duration-200"` with `style={{ transform: expandedDocId === doc.id ? 'rotate(90deg)' : undefined }}`
- Keep existing columns: title, category, current version filename, file size, date, action buttons (download + delete)
- When `expandedDocId === doc.id`, render `<VersionPanel documentId={doc.id} />` below the row

**VersionPanel sub-component (inside document-list.tsx):**
- Container: `<div className="border-l-2 border-muted ml-4 pl-4 pb-4">`
- Call `useDocumentVersions(documentId)` to fetch versions
- Loading state: 3 rows of `<Skeleton className="h-8 w-full" />`
- Each version row displays in a horizontal layout:
  - `v{versionNumber}` label (text-sm font-medium)
  - `originalFilename` (text-sm, truncate with max-w)
  - `formatFileSize(fileSize)` (text-xs text-muted-foreground) -- reuse existing helper
  - `note` if present (text-xs text-muted-foreground, italic)
  - `formatDate(createdAt)` (text-xs text-muted-foreground) -- reuse existing helper
  - If `isCurrent === true`: `<Badge variant="default">Current</Badge>`
  - If `isCurrent === false`: `<Button variant="outline" size="sm" onClick={...}>Set as Current</Button>` -- calls `useSetCurrentVersion().mutate({ documentId, versionId: version.id })` with toast "Version {N} set as current" on success
  - Download button: `<Button variant="ghost" size="icon-xs" asChild><a href={useDownloadVersionUrl(documentId, version.id)} download><Download className="size-3.5" /></a></Button>`
  - Delete button: `<Button variant="ghost" size="icon-xs" onClick={() => setDeleteVersion(version)} disabled={versions.length <= 1}><Trash2 className="size-3.5 text-destructive" /></Button>` -- disable when only 1 version remains
  - "Set as Current" button shows `<Loader2 className="size-3.5 animate-spin" />` and `disabled` during `setCurrentVersion.isPending`

**Delete confirmation:**
- Use `ConfirmDialog` with `title="Delete Version"`, `description="Version {N} will be permanently deleted. This action cannot be undone."`, `actionLabel="Delete"`
- On confirm: `deleteVersion.mutate({ documentId, versionId })` with toast "Version deleted" on success

**Upload new version section (at bottom of VersionPanel):**
- Heading or separator, then inline dropzone area
- Use `react-dropzone` (import `useDropzone` from 'react-dropzone') with the same pattern as `document-upload.tsx`:
  - `const { getRootProps, getInputProps, isDragActive } = useDropzone({ onDrop, multiple: false })`
  - Dropzone styled with `border-dashed border-2 rounded-md p-4 text-center cursor-pointer` and `border-primary bg-primary/5` when `isDragActive`
  - Text: "Upload New Version" when idle, "Drop file here" when dragging
- Optional note field: `<Input placeholder="Version note (optional)" />` below or beside the dropzone
- On file drop: build `FormData` with `file` and optional `note`, call `useCreateDocumentVersion().mutate({ documentId, formData })` with toast "New version uploaded" on success, toast "Failed to upload version. Please try again." on error
- Show `<Loader2 className="size-4 animate-spin" />` replacing dropzone content during upload
