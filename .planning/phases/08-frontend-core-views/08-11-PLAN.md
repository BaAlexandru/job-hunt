  <action>
Make dashboard recent activity cards navigate to the applications page AND auto-open the specific application's detail panel.

**1. Update dashboard/page.tsx recent activity links:**

Change the Link href in the recent activity section (around line 191) from:
`href="/applications"`
to:
`` href={`/applications?applicationId=${app.id}`} ``

This passes the application ID as a URL search parameter.

**2. Update applications/page.tsx to read the query parameter:**

At the top of `ApplicationsPage`, add `useSearchParams` from `next/navigation`:
```tsx
import { useSearchParams } from "next/navigation"
```

Inside the component, after the existing state declarations, add:
```tsx
const searchParams = useSearchParams()
```

Add a `useEffect` that reads the `applicationId` query param on mount and opens the detail panel:
```tsx
import { useEffect } from "react"
// ... (add useEffect to existing import from "react")

// Inside component body, after searchParams:
useEffect(() => {
  const appId = searchParams.get("applicationId")
  if (appId) {
    setSelectedApplicationId(appId)
  }
}, [searchParams])
```

This way, when a user clicks a recent activity card on the dashboard, they land on `/applications?applicationId=UUID` and the detail panel auto-opens for that application. The existing `ApplicationDetail` component already handles rendering when `selectedApplicationId` is non-null.

Note: The `useEffect` dependency on `searchParams` means it only fires when the URL changes, not on every render. If the user closes the panel and the URL still has the param, that's fine -- the panel just stays closed because `setSelectedApplicationId(null)` was called by `onOpenChange`.
