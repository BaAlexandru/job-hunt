Replace the `relativeTime` function (lines 30-42) with a robust version that handles negative values and properly converts to hours/days:

```typescript
function relativeTime(iso: string): string {
  const diff = Date.now() - new Date(iso).getTime()
  const absDiff = Math.abs(diff)
  const minutes = Math.floor(absDiff / 60_000)
  if (minutes < 1) return "just now"
  if (minutes < 60) return `${minutes}m ago`
  const hours = Math.floor(minutes / 60)
  if (hours < 24) return `${hours}h ago`
  const days = Math.floor(hours / 24)
  if (days < 30) return `${days}d ago`
  return new Date(iso).toLocaleDateString("en-US", {
    month: "short",
    day: "numeric",
  })
}
```

Key changes:
- Use `Math.abs(diff)` so negative timestamps (future dates or clock skew) don't produce negative output like "-2761 m ago"
- Add `minutes < 1` guard returning "just now" for very recent activity
- Rest of the logic (hours, days, date fallback) stays the same but operates on absolute values
