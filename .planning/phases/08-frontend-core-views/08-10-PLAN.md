The root cause is that the layout uses `flex` with no overflow containment, so wide kanban content pushes everything horizontally. Fix by making the sidebar fixed-position and containing overflow in the main area.

**1. Update sidebar.tsx:**
Change the aside className from:
`"hidden w-64 flex-col border-r md:flex"`
to:
`"hidden fixed inset-y-0 left-0 z-40 w-64 flex-col border-r bg-background md:flex"`

The sidebar becomes `fixed` so it stays in place regardless of horizontal scroll. Added `bg-background` to ensure it has a solid background (not transparent). Added `z-40` so it sits above scrolling content. `inset-y-0 left-0` pins it to the full height of the viewport on the left.

**2. Update layout.tsx:**
Change from:
```tsx
<div className="flex h-screen">
  <Sidebar />
  <div className="flex flex-1 flex-col">
    <Topbar />
    <main className="flex-1 overflow-auto p-3 sm:p-6">{children}</main>
  </div>
</div>
```
to:
```tsx
<div className="flex h-screen">
  <Sidebar />
  <div className="flex flex-1 flex-col md:ml-64">
    <Topbar />
    <main className="flex-1 overflow-auto p-3 sm:p-6">{children}</main>
  </div>
</div>
```

Key change: Add `md:ml-64` to the content column div. Since the sidebar is now `fixed`, it's out of document flow. The `md:ml-64` creates a left margin equal to the sidebar width (256px = w-64) so content doesn't overlap with the sidebar on desktop. On mobile (below md), sidebar is hidden so no margin needed.

**3. Update topbar.tsx:**
Change the header className from:
`"flex h-14 items-center justify-between border-b px-6"`
to:
`"sticky top-0 z-30 flex h-14 items-center justify-between border-b bg-background px-6"`

The topbar becomes `sticky top-0` so it sticks to the top of the content column when the main area scrolls vertically. Added `bg-background` for solid background and `z-30` to stay above main content but below sidebar.

These three changes together mean:
- Sidebar is fixed to the left, never moves
- Content column is offset by sidebar width on desktop
- Topbar sticks to top of content column
- Main area scrolls both vertically and horizontally without affecting sidebar/topbar

The application detail Sheet panel cannot scroll vertically on mobile. The user opens a kanban card, the Sheet slides in, but long content (Overview tab with status/dates/notes, or Notes/Interviews tabs) is cut off and cannot be scrolled to.

Root cause: The SheetContent uses `flex flex-col` internally with `h-full` (100vh). The consumer passes `overflow-y-auto` via className, but the children (SheetHeader + Tabs) are flex items that expand without constraint. The Tabs component at `className="px-4 pb-4"` has no height limit, so the flex container grows past the viewport without triggering scroll.

Fix in `frontend/components/applications/application-detail.tsx`:

1. Remove `overflow-y-auto` from the SheetContent className (the SheetContent frame should NOT scroll — only its inner content should):
```tsx
<SheetContent className="flex flex-col">
```

2. Wrap the Tabs section in a scrollable container that fills remaining space:
```tsx
<>
  <SheetHeader>
    <SheetTitle>{application.jobTitle}</SheetTitle>
    <SheetDescription>
      {application.companyName ?? "No company"}
    </SheetDescription>
    <div className="pt-1">
      <StatusBadge status={application.status} />
    </div>
  </SheetHeader>

  <div className="flex-1 min-h-0 overflow-y-auto">
    <Tabs defaultValue="overview" className="px-4 pb-4">
      {/* ... tabs content unchanged ... */}
    </Tabs>
  </div>
</>
```

Key changes:
- `flex-1` makes the scroll wrapper fill all remaining space after SheetHeader
- `min-h-0` is CRITICAL — without it, flex items have `min-height: auto` which prevents shrinking below content size, defeating overflow
- `overflow-y-auto` on this wrapper enables vertical scroll for the tabs content
- SheetHeader stays fixed at the top of the panel (not scrollable)
- The close button (absolute-positioned in SheetContent) is unaffected

This ensures:
- Sheet header (title, company, status badge) always visible
- Tab bar scrolls with content (inside the scroll wrapper)
- Long tab content (notes, interviews, documents) scrolls within the panel
- Touch scroll works on mobile because the scroll zone is a simple div, not a fixed element
