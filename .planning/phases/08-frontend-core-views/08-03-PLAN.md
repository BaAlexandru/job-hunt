**A. Replace `frontend/app/(dashboard)/companies/page.tsx`**:

"use client" page component:
1. Uses `useCompanies()` to fetch all companies
2. Uses `useJobs({ size: 1000 })` to fetch all jobs, then computes a `jobCountMap: Map<string, number>` by iterating over `jobs.content` and counting by `companyId`. This avoids N+1 queries (one query for all jobs, client-side aggregation). Example:
   ```typescript
   const jobCountMap = useMemo(() => {
     const map = new Map<string, number>()
     if (jobsData?.content) {
       for (const job of jobsData.content) {
         if (job.companyId) {
           map.set(job.companyId, (map.get(job.companyId) || 0) + 1)
         }
       }
     }
     return map
   }, [jobsData])
   ```
3. Page header with heading "Companies" (20px/600) and "Add Company" primary button
4. Renders companies as a responsive card grid:
   - `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-md` (gap 16px)
   - Each card rendered via `<CompanyCard company={company} jobCount={jobCountMap.get(company.id) || 0} ... />`
5. Click on card navigates to `/companies/{id}` detail page
6. Edit/delete actions on each card via DropdownMenu (three-dot menu icon):
   - Edit: opens CompanyForm in edit mode
   - Delete: opens ConfirmDialog with title "Delete Company", description "This will permanently delete {companyName} and unlink all associated jobs. This cannot be undone.", action "Delete"
7. Loading state: 6 skeleton cards in grid
8. Empty state: heading "No companies yet", body "Add a company to start organizing your job search."
9. Error state: "Could not load companies. Check your connection and try again."
10. CompanyForm dialog controlled by local state (`formOpen`, `editingCompany`)
