Modify `frontend/components/applications/application-board.tsx`:

1. Add imports for `useSensor`, `useSensors`, `MouseSensor`, `TouchSensor`, `KeyboardSensor` from `@dnd-kit/core`:
```typescript
import {
  useSensor,
  useSensors,
  MouseSensor,
  TouchSensor,
  KeyboardSensor,
  type DragStartEvent,
  type UniqueIdentifier,
} from "@dnd-kit/core"
```
(Note: `DragStartEvent` and `UniqueIdentifier` are already imported -- merge with existing import)

2. Create custom sensors with activation distance INSIDE the `ApplicationBoard` component, before the useMemo for columns:
```typescript
const sensors = useSensors(
  useSensor(MouseSensor, {
    activationConstraint: { distance: 5 },
  }),
  useSensor(TouchSensor, {
    activationConstraint: { delay: 200, tolerance: 5 },
  }),
  useSensor(KeyboardSensor),
)
```

The MouseSensor `distance: 5` means the pointer must move at least 5px before a drag starts. A normal click (which has <5px movement) will pass through to the card's `onClick` handler.

The TouchSensor `delay: 200, tolerance: 5` means the user must press and hold for 200ms before drag activates. A quick tap fires the click instead. The `tolerance: 5` allows slight finger movement during the hold without canceling the drag activation.

3. Pass `sensors` prop to the `<Kanban>` component. The Kanban component spreads `{...kanbanProps}` AFTER its internal `sensors={sensors}` (line 614-615 of kanban.tsx), so our prop overrides the internal one:
```tsx
<Kanban<ApplicationResponse>
  value={columns}
  onValueChange={handleValueChange}
  getItemValue={(item) => item.id}
  onDragStart={handleDragStart}
  onDragEnd={handleDragEnd}
  sensors={sensors}
  flatCursor
>
```

4. The `<KanbanItem>` keeps `asHandle` -- it still needs to be the drag handle. The activation constraint just delays the drag start, allowing click events to fire normally.

5. On the `<KanbanOverlay>` ghost card, the existing `onClick={() => {}}` no-op is fine -- it's the drag ghost and should not navigate.

No changes needed to `application-card.tsx` -- it already has `onClick` which will now fire on quick clicks because the sensor distance constraint prevents the drag from eating the event.
