Modify `frontend/components/ui/dialog.tsx` `DialogContent` function:

IMPORTANT: The dialog uses `fixed top-1/2 left-1/2 -translate-x/y-1/2` positioning — it is ALWAYS centered in the viewport. The user's complaint that "it opens somewhere down on the screen" is because long form content overflows below the viewport edge and cannot be scrolled. `scrollIntoView` does NOT help on fixed elements. The correct fix is adding `max-h-[85vh] overflow-y-auto` so the dialog content scrolls internally.

```typescript
function DialogContent({
  className,
  children,
  showCloseButton = true,
  ...props
}: React.ComponentProps<typeof DialogPrimitive.Content> & {
  showCloseButton?: boolean
}) {
  return (
    <DialogPortal>
      <DialogOverlay />
      <DialogPrimitive.Content
        data-slot="dialog-content"
        onInteractOutside={(e) => {
          // Prevent accidental close when tapping outside on mobile
          // Users must use the X button or Cancel to close
          e.preventDefault()
        }}
        className={cn(
          "fixed top-1/2 left-1/2 z-50 grid w-full max-w-[calc(100%-2rem)] max-h-[85vh] overflow-y-auto -translate-x-1/2 -translate-y-1/2 gap-4 rounded-xl bg-background p-4 text-sm ring-1 ring-foreground/10 duration-100 outline-none sm:max-w-sm data-open:animate-in data-open:fade-in-0 data-open:zoom-in-95 data-closed:animate-out data-closed:fade-out-0 data-closed:zoom-out-95",
          className
        )}
        {...props}
      >
        {children}
        {showCloseButton && (
          <DialogPrimitive.Close data-slot="dialog-close" asChild>
            <Button
              variant="ghost"
              className="absolute top-2 right-2"
              size="icon-sm"
            >
              <XIcon />
              <span className="sr-only">Close</span>
            </Button>
          </DialogPrimitive.Close>
        )}
      </DialogPrimitive.Content>
    </DialogPortal>
  )
}
```

Key changes:
- Added `max-h-[85vh] overflow-y-auto` to the className — this caps dialog height at 85% of viewport and enables internal scrolling. Long forms (job creation with 15+ fields) will scroll within the dialog instead of overflowing off-screen.
- Added `onInteractOutside` with `e.preventDefault()` so clicking/tapping the overlay does NOT close the dialog — users must use the X button or Cancel/Close button in the footer.
- NO `onOpenAutoFocus` with `scrollIntoView` — this was the wrong fix for fixed-positioned elements. Removed entirely.
- The `{...props}` spread remains AFTER `onInteractOutside` so consumers can override if needed.
