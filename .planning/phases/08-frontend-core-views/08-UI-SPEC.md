## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, kanban card inner padding |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, filter bar padding, panel inner margins |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical padding, empty state spacing |
| 3xl | 64px | Not used in this phase |

Exceptions: Kanban column width fixed at 280px. Slide-over panel width at 480px (sm:max-w-lg). Touch targets minimum 44px for mobile drag handles and action buttons.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 600 (semibold) | 1.5 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Usage notes:
- **Body (14px/400):** Table cells, card body text, form inputs, notes content, filter bar inputs
- **Label (12px/600):** Column headers, status badges, count badges, metadata labels (applied date, next action), table column headers, filter dropdown labels
- **Heading (20px/600):** Page titles (Applications, Companies, Jobs, Documents, Dashboard), slide-over panel title, modal dialog titles
- **Display (28px/600):** Dashboard metric values only (e.g. total application count)

## Color

All colors use the existing OKLCH token system defined in `globals.css`. No new color tokens are introduced.

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | Page background, panel backgrounds |
| Secondary (30%) | `--card` / `--secondary` | oklch(1 0 0) / oklch(0.97 0 0) | Cards, kanban columns, sidebar, table rows |
| Accent (10%) | `--primary` | oklch(0.205 0 0) | See reserved-for list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | Delete buttons, archive confirmations |

Accent (`--primary`) reserved for:
- Primary CTA buttons ("New Application", "Add Company", "Add Job", "Upload Document")
- Active tab indicator (Board/List toggle, detail panel tabs)
- Kanban drag-over column border highlight
- Active sidebar nav item
- Pagination current page indicator

### Status Badge Colors

Status badges use semantic coloring via Tailwind utility classes. These are the only non-token colors in the phase:

| Status | Background | Text | Rationale |
|--------|-----------|------|-----------|
| INTERESTED | `bg-blue-100 text-blue-800` | dark:blue-900/bg-blue-400 | Cool, neutral starting point |
| APPLIED | `bg-indigo-100 text-indigo-800` | dark variant | Active, in-progress |
| PHONE_SCREEN | `bg-violet-100 text-violet-800` | dark variant | Advancing |

| REJECTED | `bg-red-100 text-red-800` | dark variant | Negative outcome |

## Interaction Contracts

### Kanban Board

| Interaction | Behavior |
|-------------|----------|
| Drag start | Card lifts with subtle shadow; valid drop columns retain normal style; invalid columns dim to 40% opacity with `pointer-events-none` |
| Drag over valid column | Column border changes to `--primary` color; slight background tint |
| Drop on valid column | Optimistic status update; toast on success; revert on error |
| Drop on invalid column | Card snaps back to original position; no API call |
| Card click | Opens ApplicationDetail Sheet from right side (480px wide) |
| Column header | Shows status name + count badge (e.g. "Applied (3)") |
| Horizontal scroll | Overflow-x-auto on board container; CSS scroll-snap on mobile |
| "+ New Application" button | Opens ApplicationForm Dialog (centered modal) |
| Archive toggle | Checkbox above board: "Show archived" -- toggles archived applications visibility |

### List/Table View

| Interaction | Behavior |
|-------------|----------|
| Column header click | Toggles sort direction (asc/desc) with sort indicator arrow |
| Row click | Opens ApplicationDetail Sheet (same component as kanban) |
| Inline status dropdown | Shows only valid transitions for that row's current status |
| Status change via dropdown | Optimistic update, same mutation as kanban drag |
| Filter bar | Inline above table: Status multi-select, Company combobox, Date range picker, Search input with debounce (300ms) |
| Pagination | Numbered pages below table; 20 rows per page; shows "Page X of Y" |
| Empty filtered state | Shows filtered empty state copy (not global empty state) |

### Slide-Over Detail Panel (Sheet)

| Interaction | Behavior |
|-------------|----------|
| Open | Slides in from right, 480px wide, board/list visible underneath with overlay |
| Close | Click overlay, press Escape, or click X button |
| Tab: Overview | Application metadata, status dropdown, dates, quick notes |
| Tab: Notes | Scrollable notes list with add/edit/delete; newest first |
| Tab: Interviews | Interview rounds list with add/edit/delete |
| Tab: Documents | Linked documents list with upload and download buttons |
| Nested dialog | Edit/delete from within panel opens Dialog on top of Sheet (portal outside Sheet) |

### CRUD Form Dialogs

| Interaction | Behavior |
|-------------|----------|
| Open | Centered modal via shadcn Dialog; overlay dims background |
| Validation | Inline field-level errors via React Hook Form + Zod; shown on blur and submit |
| Submit (create) | Mutation via TanStack Query; toast on success; dialog closes; list refreshes |
| Submit (edit) | Same pattern; form pre-populated with existing data |
| Cancel | Dialog closes; no save; form state discarded |

### Document Upload

| Interaction | Behavior |
|-------------|----------|
| Dropzone | Dashed border area; border changes to `--primary` on drag-over |
| File validation | Accept PDF and DOCX only; reject others with toast error |
| Upload progress | Show uploading state on the file entry (spinner or progress bar) |
| Upload complete | Toast success; document appears in list; query invalidated |

### Loading States

| View | Loading Pattern |
|------|-----------------|
| Kanban board | 8 Skeleton columns with 2-3 Skeleton cards each |
| Data table | Skeleton rows (6 rows) with column-width-matched Skeleton cells |
| Detail panel | Skeleton blocks for header + tab content |
| Dashboard cards | Skeleton metric cards (4 cards) |
| Company grid | Skeleton cards (6 cards in grid) |
| Document list | Skeleton table rows |

### Dice UI Kanban Vetting

The `@diceui/kanban` block is installed via the shadcn CLI (`npx shadcn@latest add @diceui/kanban`). This is a third-party registry component.

**Vetting procedure:** Before installation, run `npx shadcn@latest view @diceui/kanban` and scan for:
- `fetch(`, `XMLHttpRequest`, `navigator.sendBeacon` -- network access
- `process.env` -- environment variable access
- `eval(`, `Function(`, `new Function` -- dynamic code execution
- Dynamic imports from external URLs
- Obfuscated variable names

**Status:** Pending -- executor must run the view command and record the result before installing. If any flags are found, the executor must flag for developer review before proceeding.
