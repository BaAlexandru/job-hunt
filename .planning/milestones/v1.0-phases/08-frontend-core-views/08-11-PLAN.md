Fix three issues with the status dropdown in the data table:

1. **Scroll blocking fix (PRIMARY):** The dropdown blocks ALL horizontal table scroll because Radix DropdownMenu is `modal` by default — it renders a full-viewport invisible overlay that captures ALL pointer events. Add `modal={false}` to the DropdownMenu:
   ```tsx
   <DropdownMenu modal={false}>
   ```
   This removes the overlay and allows background interaction (scrolling) while the dropdown is open. The dropdown will close on outside click or scroll, which is the expected behavior for inline table dropdowns.

2. **Dropdown width truncation:** In the `StatusCell` component, change the `DropdownMenuContent` from:
   `<DropdownMenuContent align="start">`
   to:
   `<DropdownMenuContent align="start" className="min-w-[180px]" sideOffset={4} collisionPadding={8}>`

   This ensures the dropdown is at least 180px wide (enough for "Phone Screen" without truncation), offset by 4px from trigger, and repositions if it would extend beyond the viewport edge.

NOTE: Do NOT add `size: 140` to the column definition — TanStack Table's `size` property has no effect in this DataTable implementation since the component doesn't read column sizes for rendering. The min-w-[180px] on the dropdown content is sufficient.
  </action>

Make dashboard recent activity cards navigate to the applications page AND auto-open the specific application's detail panel.

**1. Update dashboard/page.tsx recent activity links:**

Change the Link href in the recent activity section (around line 191) from:
`href="/applications"`
to:
`` href={`/applications?applicationId=${app.id}`} ``

This passes the application ID as a URL search parameter.

**2. Update applications/page.tsx to read the query parameter and clean up on close:**

At the top of `ApplicationsPage`, add `useSearchParams` and `useRouter` from `next/navigation`:
```tsx
import { useSearchParams, useRouter } from "next/navigation"
```

Add `useEffect` to the existing import from `"react"`:
```tsx
import { useState, useEffect } from "react"
```

Inside the component, after the existing state declarations, add:
```tsx
const searchParams = useSearchParams()
const router = useRouter()
```

Add a `useEffect` that reads the `applicationId` query param on mount and opens the detail panel:
```tsx
useEffect(() => {
  const appId = searchParams.get("applicationId")
  if (appId) {
    setSelectedApplicationId(appId)
  }
}, [searchParams])
```

Update the `ApplicationDetail` `onOpenChange` handler to clear the URL param when the panel is closed:
```tsx
<ApplicationDetail
  applicationId={selectedApplicationId}
  open={selectedApplicationId !== null}
  onOpenChange={(open) => {
    if (!open) {
      setSelectedApplicationId(null)
      // Clear the applicationId query param so reopening the page doesn't re-trigger the panel
      if (searchParams.has("applicationId")) {
        router.replace("/applications", { scroll: false })
      }
    }
  }}
/>
```

This way:
- Clicking a dashboard card navigates to `/applications?applicationId=UUID` and auto-opens the detail panel
- Closing the panel clears the URL param so the panel doesn't re-open on navigation back
- `router.replace` (not push) avoids adding a history entry for the param cleanup
- `{ scroll: false }` prevents the page from jumping on URL change
  </action>
